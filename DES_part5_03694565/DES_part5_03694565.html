 Task 5.3.2: Confidence Interval Width¶

In [5]:
p = 0.9
S = 4.0
bp = ((1-p)*(p**(S+1)))/(1-(p**(S+2)))
print (bp)

0.1260225499883686
  • The analytic result is 0.1260 as can be seen above calculation. The main reason of such differences is the transient phase of the simulation. This can be easily seen if we compare the 100s and 1000s simulations in the task_5_2_1(). 1000s simulation result is much close to the actually blocking probablity since the transient effect can be tolerated during the simulation.
  • There is not that much differences between 100 and 1000 batch sizes but 1000 batch size is more near to actually probability. Because if we get 100 batch size than we can have more samples from the transient phase which make it diffucult compensate during simulation.

 Task 5.3.3: Confidence Interval Width¶

  • The values written on the graph may not be clear to read, but it follows the rule of printing values as written above the plots. Observations;
  • Confidence interval is larger for small simulation time(e.g. 100s)
  • Alpha value is important to understand how many of the red lines cover the actual utilization value.
  • In the longer simulations, the difference between actual rho and observed rho become lower.
  • For the Rho = 0.9, observed rho value is different(lower) than the actuall rho value. This effect can be seen especially for lower simulation time.

Task 5.3.5: Bootstrapping¶

  • If the number of samples are small, then bootstrap method can be used to resample the same data multiple times and treating the sample as an approximation of the true distribution of data.
  • If we know the distribution of the samples are from true distribution, then they might become the same.
